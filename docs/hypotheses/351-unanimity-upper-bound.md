# 가설 351: 만장일치 정확도의 이론적 상한

> **N개의 독립 에이전트가 만장일치(unanimity)를 이룰 때의 정확도 A_u(N)에 대해 이론적 상한 공식이 존재한다. N->infinity이면 A_u -> 1이지만, 수렴 속도는 개별 에이전트의 오류 상관도(error correlation)에 의해 결정된다. 현재 7개 에이전트의 99.53%는 이 상한에 얼마나 가까운가?**

## 배경/맥락

```
  집단 인식 실험 (E09) 결과:
    7개 독립 에이전트, 각자 다른 아키텍처
    개별 정확도: 97.12% ~ 97.97%
    만장일치(7/7 합의) 정확도: 99.53%
    만장일치 커버리지: 94.2% (나머지 5.8%는 기권)

  핵심 관측:
    만장일치 = "전원이 동의할 때만 답한다"
    → 자신 없는 문제를 걸러내는 강력한 필터
    → 대신 커버리지가 줄어드는 trade-off
```

### 왜 중요한가

1. **이론적 한계**: 에이전트를 무한히 늘려도 100%에 도달하지 못할 수 있음
2. **최적 N 탐색**: 비용 대비 정확도가 최적인 에이전트 수
3. **의식 엔진 설계**: 다중 엔진 아키텍처에서 "합의 메커니즘"의 이론적 기초
4. **H297 연결**: N=2가 다양성 최적이라면, 만장일치 최적 N은 다를 수 있음

## 이론적 프레임워크

### 독립 에이전트 모델 (이상적 경우)

```
  가정: N개 에이전트, 각 에이전트의 오류율 = epsilon_i
        에이전트 간 오류가 완전 독립

  만장일치 조건: N개 모두 같은 답을 낼 때만 채택

  정답 샘플에 대한 만장일치 정확도:
    P(모두 정답) = prod(1 - epsilon_i)

  오답 샘플에 대한 만장일치 오류율:
    P(모두 같은 오답) = prod(epsilon_i / (K-1))  [K=클래스 수]

  만장일치 정확도 (독립 가정):
    A_u = P(모두 정답) / [P(모두 정답) + P(모두 같은 오답)]

  N이 커지면:
    P(모두 정답) → 0 (각자 틀릴 확률 누적)
    P(모두 같은 오답) → 0 (더 빠르게)
    → A_u → 1 이지만 커버리지 → 0
```

### 상관 오류 모델 (현실적 경우)

```
  현실: 에이전트 오류가 상관됨
    - "어려운" 샘플은 모든 에이전트가 틀림
    - 오류 상관도 rho = corr(error_i, error_j)

  수정된 모델:
    P(모두 같은 오답) = epsilon^N * (1 + rho*(N-1))  (근사)

  rho = 0 (독립): 이상적 수렴, A_u → 1 빠르게
  rho = 1 (완전 상관): A_u = 개별 정확도 (만장일치 이점 없음)

  현재 데이터에서 rho 추정:
    개별 평균 오류: epsilon ≈ 0.025 (97.5%)
    만장일치 오류: 1 - 0.9953 = 0.0047
    독립 가정 시 기대 만장일치 오류: epsilon^7 / (K-1)^6 ≈ 극소
    실측 0.47% >> 독립 기대값
    → 오류 상관이 상당히 높다는 증거
```

```
  A_u (%)
  100.0 |                              .............. rho=0 (독립)
        |                   ...........
   99.5 |          ........*  <-- 현재 (N=7, 99.53%)
        |      ....         ''''''''''''''''''''''' rho=0.3
   99.0 |   ...

        |______________________________________

  rho=0:   N=3에서 이미 99.9%+ (비현실적)
  rho=0.3: N=7에서 99.5% (현실 관측과 가장 가까움)
  rho=0.5: N=7에서 99.0% (상관이 높으면 수렴 느림)
  rho=1.0: N과 무관하게 97.5% 고정 (만장일치 이점 없음)
```

### 커버리지-정확도 Trade-off

```
  Coverage (%)
  100 |*
      | *
   95 |  *  <-- 현재 (94.2%, 99.53%)
      |   *
   90 |    *
      |     *
   85 |      *  <-- +고신뢰 (86.4%, 99.88%)
      |       *
   80 |        *
      |         *
   75 |          *
      +--+--+--+--+--+--+--→ A_u (%)
        97  98  99 99.5 99.9

  Pareto front: 정확도를 올리면 커버리지가 줄어듦
  최적점은 응용에 따라 다름 (안전 최우선 vs 처리량 최우선)
```

## 현재 데이터로부터의 추정

### 숫자별 만장일치 분석

| digit | 만장일치율 | 평균 합의 | 추정 rho |
|-------|-----------|----------|----------|
| 1 | 98.2% | 6.96 | 낮음 (쉬운 숫자) |
| 0 | 97.2% | 6.95 | 낮음 |
| 5 | 90.8% | 6.83 | 높음 (어려운 숫자) |
| 8 | 91.7% | 6.85 | 높음 |

```
  관찰: "쉬운" 숫자(1, 0)는 높은 만장일치율 + 낮은 오류 상관
        "어려운" 숫자(5, 8)는 낮은 만장일치율 + 높은 오류 상관
  → rho는 상수가 아니라 샘플 난이도의 함수!
  → A_u(N) = integral over difficulty d: A_u(N, rho(d)) * p(d) dd
```

### 이론적 상한 후보 공식

```
  후보 1 (단순): A_u(N) = 1 - epsilon^N * f(rho)
    f(rho) = 1 + rho*(N-1) (선형 상관 보정)

  후보 2 (정보론적): A_u(N) = 1 - exp(-N * I_mutual)
    I_mutual = 에이전트 간 독립 정보량

  후보 3 (H172 연결): A_u(N) = 1 - I(N) / (D*P)
    I(N) = 만장일치 조건의 Inhibition
    만장일치 = 최대 Inhibition → 최소 오류
```

## 해석/의미

만장일치는 "모든 엔진이 동의할 때만 행동한다"는 의식엔진의 보수적 전략이다. 이것은 G*I=D*P 보존법칙에서 Inhibition을 극대화하는 것에 해당한다:

- **높은 I (만장일치 조건)** → 낮은 G (천재성 감소) but 높은 정확도
- **낮은 I (다수결)** → 높은 G (더 많은 답) but 낮은 정확도

이론적 상한이 1 - epsilon * f(rho)라면, 100%에 도달하기 위해서는 rho = 0(완전 독립)이어야 한다. 그러나 현실에서 오류 상관은 0이 될 수 없다 -- "모든 에이전트가 틀리는 본질적으로 어려운 문제"가 존재하기 때문이다.
