# H-ROB-5: Mitosis = Swarm Fission

## 가설

> TECS-L의 Mitosis (세포분열) 메커니즘은 swarm robotics에서의 군집 분열에
> 대응한다. 최적 분열 수 N=2 (sigma_-1(6) = 2)가 per-robot efficiency를
> 최대화하며, 이는 이진 분열의 보편적 최적성을 반영한다.

## 배경

TECS-L에서 Mitosis(H-297)는 하나의 엔진이 임계점에 도달했을 때 두 개로
분열하는 메커니즘이다. sigma_-1(6) = 2가 최적 분열 수를 예측한다.

Swarm robotics에서 군집 분열은 대규모 영역을 효율적으로 커버하기 위한
전략이다. 문제는: 몇 개의 하위 그룹으로 나누는 것이 최적인가?

관련 가설: H-297 (Mitosis), H-172 (G*I=D*P 보존법칙)

## 실험 설정

```
Area: 100 x 100 (단위 영역)
Grid: 20 x 20 = 400 cells (커버리지 측정)
Steps: 500 (시뮬레이션 스텝)
Sense radius: 15.0 (이웃 감지 반경)
Robot speed: 1.0/step
N values: [1, 2, 3, 4, 5, 6, 8, 10, 12]
Trials: 5 (평균)

각 N에 대해:
  - N개의 수직 strip으로 영역 분할
  - 각 strip에 1 robot 배치
  - Robot은 자기 strip 내 미탐색 cell로 이동
  - Coordination overhead = 감지 범위 내 통신 이벤트

지표:
  - Coverage: 탐색된 cell 비율
  - Overhead: 정규화된 통신 이벤트 수
  - Efficiency: coverage / N (per-robot)
  - Net benefit: coverage - 0.5 * overhead * N
```

## 해석

1. **Per-robot efficiency**: N=1과 N=2가 동률(0.3600)이다.
   이는 2개의 robot이 서로 간섭 없이 독립적으로 작동할 때
   개별 효율이 유지됨을 의미한다. N >= 3부터 효율이 감소하기 시작한다.
   sigma_-1(6) = 2 예측은 **부분적으로 확인**: N=2가 효율을 잃지 않는
   최대 분열 수이다.

2. **Net benefit**: N=4 = tau(6)이 최적이다. 이는 커버리지(1.0)를
   달성하면서 overhead가 최소인 지점이다. tau(6) = 4가
   "완전 커버리지의 최소 단위"라는 해석이 가능하다.

3. **Overhead 구조**: N=1,2에서 overhead = 0 (로봇 간 간섭 없음),
   N=3에서 0.09로 급증, N >= 4에서는 strip이 좁아져 간섭이 적어지다가
   N >= 8에서 다시 증가한다. 이는 감지 반경과 strip 폭의 관계이다.

4. **수렴 포인트**: N >= 4에서 coverage = 1.0으로 수렴한다.
   4 = tau(6)이 "충분한" 분할의 임계점이다.

## 제한사항

- 1 robot per group은 현실적이지 않음 (실제 swarm은 group당 다수)
- Vertical strip 분할은 naive (Voronoi 등이 더 효율적)
- Overhead 모델이 단순 (감지 반경 내 쌍 카운팅)
- 5회 평균은 통계적으로 불충분
- sigma_-1(6) = 2 예측이 정확히 확인되지 않음 (N=1과 동률)
