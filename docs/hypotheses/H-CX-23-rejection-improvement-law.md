# H-CX-23: 거부 개선 법칙 — improvement ≈ ln(K) × √(error_rate)

> **확신 거부의 개선폭이 ln(K)×√(100-base)를 따른다. 여기서 K=클래스 수(10), ln(10)=2.30. 이것은 C57(C48/τ(6)≈ln(10))과 같은 상수이며, 10클래스 분류의 "정보 용량"을 반영한다.**

## 실측

```
  3데이터셋, 거부 90% 결과:
  Dataset    base    100-base   √(100-base)  improvement  imp/√(err)
  ────────  ──────  ─────────  ───────────  ───────────  ──────────
  MNIST      97.72    2.28       1.51         1.48        0.980
  Fashion    87.99   12.01       3.47         9.81        2.831
  CIFAR      51.55   48.45       6.96        15.18        2.181

  피팅: improvement = 2.31 × (100-base)^0.49
  coefficient = 2.31 ≈ ln(10) = 2.303 (오차 0.4%!)
  exponent = 0.49 ≈ 1/2 = 골든존 상한 = 리만 임계선!
```

## 수학적 의미

```
  improvement ≈ ln(K) × √(error_rate)

  ln(K): K=10 클래스의 정보 엔트로피 (균등 분포)
    = log₂(10) × ln(2) ≈ 3.32 × 0.693 = 2.303
    = "10개 선택지의 최대 불확실성"

  √(error_rate): error의 제곱근
    = "표준편차 스케일" (CLT에 의해 √N 스케일링)
    = 거부로 제거 가능한 오류의 "폭"

  종합: "거부가 줄일 수 있는 오류" = "정보 용량" × "오류 폭"
```

## C57과의 연결

```
  C57: C48/τ(6) ≈ ln(10)
    C48 = -9.25pp (장력 인과 효과)
    τ(6) = 4 (약수 개수)
    C48/τ(6) = 9.25/4 = 2.3125 ≈ ln(10)

  H-CX-23: improvement coefficient = 2.3117 ≈ ln(10)

  → 같은 상수(ln(10))가 두 곳에서 등장:
    1. 장력 인과 효과 / 약수 개수 = ln(K)
    2. 거부 개선 계수 = ln(K)
    → 10클래스 분류에서 ln(10)은 "근본 상수"?
```

## 주의

```
  3점 피팅: 자유도 1 (2 파라미터에 3 데이터점)
  MNIST가 outlier (0.98 vs 2.18, 2.83)
  → K=10 아닌 다른 클래스 수에서 검증 필요
  → K=2, K=5, K=26에서 coefficient가 ln(K)인지 확인
```

## K 변화 검증 (2026-03-24)

```
  K vs coefficient:
  K     coeff   ln(K)   ratio
  ───   ─────   ─────   ─────
  2     0.22    0.69    0.31  (base=99.95%, 개선 거의 없음)
  5     0.76    1.61    0.47  (base=99.42%, 개선 작음)
  10    2.31    2.30    1.00  ← 일치!

  → K=2, K=5에서 불일치!
  → 이유: 기저 정확도가 너무 높아 개선 여지 없음
  → ln(K) 관계는 K=10에서만 성립 → 우연일 가능성

  coefficient ≈ 2.31 = ln(10)은 "10클래스의 정보 용량" 때문이 아니라
  해당 데이터셋들의 기저 정확도와 오류 분포의 우연의 일치?
```

## 상태: ⚠️ 약화 (K=10에서만 일치, K=2,5 불일치 → 우연 가능성)
