# EVOL-003: DNA 염기 = τ(6) = 4

> **가설**: DNA의 4개 염기(A, T, G, C)는 τ(6) = 4이며, 이는 정보 저장의 최적 알파벳 크기이다.

## 왜 4개인가?

```
알파벳 크기 vs 정보 효율:

  2-base: 2 bits/base, 단순하지만 긴 서열 필요
  3-base: log₂3 = 1.58 bits/base, 불균형
  4-base: 2 bits/base, 짝수 → 상보적 쌍 = φ(6) = 2
  5-base: 비효율적 (홀수, 상보쌍 불가)
  6-base: 과잉, 복제 오류 증가

  → 4 = τ(6)가 최적:
     ✓ 2 bits/position (깨끗한 이진)
     ✓ 2 = φ(6) 상보적 쌍 가능
     ✓ 적절한 여유도
```

| 속성 | 값 | 연결 |
|------|------|------|
| 염기 종류 | 4 | τ(6) |
| 퓨린 | 2 (A,G) | φ(6) |
| 피리미딘 | 2 (T,C) | φ(6) |
| 수소결합 (A-T) | 2 | φ(6) |
| 수소결합 (G-C) | 3 | P₁/2 |
