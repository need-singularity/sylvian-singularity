# H-AI-10: BPE Tokenizer 최적 Vocab 크기와 6의 거듭제곱

> **가설**: BPE tokenizer의 최적 vocab 크기가 6^k 근처에서 효율 극대화된다.

## 배경
- GPT-2: 50257 ≈ 6^6×0.7, GPT-4: 100K, LLaMA: 32K
- 6^5=7776, 6^6=46656, 6^7=279936
- 가설이 약함: 실제 vocab 크기는 2^k에 더 가까움 (32K=2^15)

## 검증 방향
1. [ ] vocab sweep: 1K..100K에서 downstream task 성능
2. [ ] 6^k vs 2^k 근처에서 성능 비교

## 현실적 평가: 매우 약함 (★). 2^k 편향이 더 강함.
