## Background

Kolmogorov complexity K(x) = length of the shortest program that outputs x.
While K(x) is uncomputable in general, we can bound it. For integers,
K(n) ≈ log₂(n) + O(log log n). But STRUCTURED numbers can be shorter.

## Why n=6 is Maximally Compressible

```
  Descriptions of 6:
    "2·3"          — 2 symbols (product of consecutive primes)
    "3!"           — 2 symbols (factorial of smallest odd prime)
    "1+2+3"        — 3 symbols (triangular number T(3))
    "P₁"           — 2 symbols (first perfect number)

  Compare other composites:
    4 = 2² (2 symbols, but prime power = less structural)
    8 = 2³ (2 symbols, prime power)
    9 = 3² (2 symbols, prime power)
    10 = 2·5 (2 symbols, but non-consecutive primes)
    12 = 2²·3 (3 symbols)

  6 = 2·3 is THE simplest composite with distinct prime factors.
  It's the only number expressible as:
    p₁·p₂ = T(p₂) = p₂! = P₁ simultaneously.
```

## Consciousness and Compression

```
  Kolmogorov complexity of consciousness:
    A conscious system needs STRUCTURE (not a prime = too simple)
    But also COMPRESSIBILITY (not random = too complex)

  n=6 = minimum K(n) among structured composites:
    Structured: τ=4 divisors, ω=2 prime factors
    Compressible: "2·3" = shortest possible non-trivial factorization
    Rich: σ=12, φ=2, sopfr=5 — maximum info per bit of description

  Information density = (number of derived constants) / K(n)
    n=6: ~200 characterizations / ~3 bits ≈ 67 characterizations/bit
    n=28: ~50 characterizations / ~5 bits ≈ 10 characterizations/bit
    → n=6 has 6.7× higher information density!
```

## Limitations

- Kolmogorov complexity is uncomputable; we use description length as proxy
- "Simplest composite" is informal — depends on the description language
- The information density comparison is approximate
