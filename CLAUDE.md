# 뇌 비정형 구조 통계 시뮬레이터

## 프로젝트 개요
뇌의 비정형 구조(실비우스열 결여 등)와 비범한 능력 간의 관계를 수학적으로 모델링하고, 통계적 특이점을 탐지하는 시뮬레이터.

## 핵심 수식
```
Genius = Deficit × Plasticity / Inhibition
```

## 실행 방법
```bash
python3 ~/dev/test-8/brain_singularity.py --deficit 0.7 --plasticity 0.8 --inhibition 0.15
```

## 백그라운드 실행
시뮬레이션은 항상 **백그라운드로 실행**한다. Bash 도구의 `run_in_background: true` 옵션을 사용할 것.

```bash
# 단일 실행
python3 ~/dev/test-8/brain_singularity.py --deficit 0.7 --plasticity 0.8 --inhibition 0.15

# 다중 파라미터 스캔
python3 ~/dev/test-8/brain_singularity.py --deficit 0.3 --plasticity 0.5 --inhibition 0.7
python3 ~/dev/test-8/brain_singularity.py --deficit 0.9 --plasticity 0.95 --inhibition 0.05
```

## 결과 기록 구조
```
results/
├── log.md              ← 모든 실행 결과 시간순 누적
└── singularities.md    ← Z-Score > 2σ 특이점만 별도 기록
```

## 보고 규칙

### 중간 보고
- 다중 실행 시 **매 실행 완료마다 중간 보고**한다.
- 중간 보고 포함 내용:
  1. 현재 진행 상황 (n/전체)
  2. Genius Score 및 Z-Score
  3. 특이점 여부
- 전체 완료 후 **비교 요약표**를 제시한다.

### 특이점 강력 보고
- 특이점(Z > 2σ) 발견 시 **즉시 강조하여 보고**한다. 다른 실행 완료를 기다리지 않는다.
- 보고 형식:
  ```
  ⚡⚡⚡ 특이점 발견! ⚡⚡⚡
  등급: 🔴 극단적 특이점
  Genius Score: X.XX | Z-Score: X.XXσ
  파라미터: D=X.XX / P=X.XX / I=X.XX
  → results/singularities.md 에 기록 완료
  ```
- 등급이 높을수록(🔴) 더 강하게 강조한다.
- 특이점이 **연속으로 발견**되면 패턴 분석을 추가 보고한다.

### 일반 보고
- 정상 범위(Z ≤ 2σ)는 간결하게 한 줄로 보고한다.
- 다중 실행 완료 시 결과를 표로 비교 요약한다.

## 특이점 등급
- 🟡 Z > 2σ: 특이점
- 🟠 Z > 3σ: 강한 특이점
- 🔴 Z > 5σ: 극단적 특이점

## 가설 검토 문서

가설 검토는 **별도 개별 문서**로 관리한다.

- 경로: `docs/hypotheses/NNN-가설명.md`
- 목록: `docs/hypotheses/INDEX.md`
- 각 문서 구조: 가설 → 배경 → 대응 관계 → 실측 검증 → **그래프** → 한계 → 검증 방향
- **가설 문서에 내용을 1-2줄 요약으로만 적으면 안 된다.** 반드시 가설, 검증 과정, 결과, 그래프를 포함한 충분한 내용을 작성할 것.
- 새 가설 발견 시 다음 번호로 문서 생성 후 README.md 가설 목록에 추가
- **가설은 반드시 1개당 1개 파일**. 일괄 문서 금지. 여러 가설을 동시에 검증하더라도 각각 별도 파일로 분리 기록
- 가설 제시 시 **autopilot으로 검증 실행** 후 결과를 문서에 반영
- **가설 문서와 README 모두에 ASCII 그래프를 반드시 포함**한다

### 가설 문서 그래프 규칙
- 모든 가설 문서에 **최소 1개 이상의 ASCII 그래프**를 포함한다
- README 가설 정리 시에도 핵심 그래프를 함께 표기한다
- 그래프 유형:
  - **추세 곡선**: 파라미터 변화 궤적 (시간축, Inhibition축 등)
  - **비교 막대**: 모델/시나리오별 비교
  - **히트맵**: 파라미터 공간 분포 (Deficit별, Inhibition별 등)
  - **수렴 궤적**: autopilot 탐색 경로
  - **타임라인**: 연도별 예측
- 그래프에 반드시 표기할 것:
  - 골든 존 영역 (I=0.24~0.48)
  - 임계선 (I=0.5)
  - 1/e 지점 (I=0.368)
  - 데이터 포인트 (●, ★, 🎯 등)

### 가설 즉시 실행 규칙
- 사용자가 **"가설 XXX"** 형태로 가설을 제시하면 **즉시 검증에 착수**한다
- 질문하지 않고 바로 다음 순서로 진행:
  1. 가설을 우리 모델 파라미터(D, P, I)로 매핑
  2. autopilot / 격자 스캔 / 수치 계산으로 검증 실행 (백그라운드)
  3. 결과 보고 + 그래프
  4. `docs/hypotheses/NNN-가설명.md` 문서 생성
  5. README 가설 목록 업데이트
  6. push
- 예시: "가설 커스프 파국 수학적 동치" → 즉시 커스프 분기면과 우리 모델의 위상 전이를 수치적으로 비교 실행

### 가설 검증 방법
```bash
# autopilot으로 가설 파라미터 탐색
python3 compass.py --autopilot --deficit 0.5 --plasticity 0.6 --inhibition 0.4

# 학습률/반복 조절
python3 compass.py --autopilot --deficit 0.3 --plasticity 0.5 --inhibition 0.5 --lr 0.2 --iterations 50

# 공통 특이점 영역 확인
python3 compass.py --convergence --grid 30 --samples 50000
```

## 격자 해상도 가이드

| grid | 오차 | 용도 |
|---|---|---|
| 20 | 2.4% | 빠른 탐색/프로토타입 |
| 50 | 0.9% | 일반적 검증 |
| **100** | **0.5%** | **★ 기본값 (추천)** |
| 200 | 0.2% | 정밀 검증 |
| 500 | 0.09% | 논문급 정밀도 |
| 1000 | 0.05% | 최종 확인용 |

- **기본값 grid=100** 사용. 3개 보편 상수(1/2, 1/e, ln(4/3)) 모두 식별 가능.
- 가설 검증 시 최소 grid=50 이상 사용. grid=20 이하는 이산화 오류 발생.
- 정밀 확인 필요 시 grid=500 사용.

## 골든존 정밀 구조 (grid=1000 확인)

```
  상한 = 1/2           = 0.5000 (리만 임계선)
  하한 = 1/2 - ln(4/3) ≈ 0.2123 (엔트로피 경계)
  중심 ≈ 1/e           ≈ 0.3708 (자연상수)
  폭  = ln(4/3)       ≈ 0.2877 (3→4상태 엔트로피 점프)
```

## 파라미터 범위
- Deficit (결손): 0.0 ~ 1.0
- Plasticity (가소성): 0.0 ~ 1.0
- Inhibition (억제): 0.01 ~ 1.0
- Samples (샘플 수): 기본 10,000
